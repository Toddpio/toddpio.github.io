Bristol-Meyers Squibb

Data visualization + mobile UI.




Background

Opdivo’s digital AOR failed to adapt the clinical data for mobile users by placing charts designed for print directly into Opdivohcp.com.


My role

  • Research
  • •
  • UX Design
  • •
  • UI Design


01:  Research

Stakeholder interviews

I met with internal and client stakeholders to gain an understanding of the business needs.

Design evaluation

I surveyed the existing design to assess how the users would ideally interact with the data.

Brand constraints

The new design would need to adhere to existing—brand Colors, fonts, and established UI styles and systems.




02:  UX

Problem

Vital data charts initially created for a print environment were directly placed on the website, which was OK for the desktop version, but completely unusable for the mobile version.



Evaluation

It was instantly apparent that a complete redesign was necessary.



Solutiuon

Start from scratch using a mobile-first mindset.



Process

Stripped the content down to the essential elements and built up from there.



Final design & motion graphics

The design was much more user freindly and allowed for opportunities to delight.



03:  UI

Final designs

The approach proved to be very successful, with a near 100% increase in readability and usability. And a thrilled client.
